Critical Knowledge · Product Pivot

The data changed. The architecture follows.

The original PDFs described daily mandi forecasting with Prophet and Agmarknet CSVs. The actual checked-in data is season-wise commodity reports. Build around reality, not the plan.

What the data actually is

Four season-wise CSVs (2022-23 through 2025-26), already normalized to season_report_summary.json by Member B. Fields: commodity_group, commodity, msp, kharif_price, kharif_arrival_tonnes, rabi_price, rabi_arrival_tonnes. 14 commodities across Cereals, Pulses, Oil Seeds, Fibre Crops. No mandi. No daily dates. No Agmarknet.

What this means for backend scope

No Prophet. No Supabase. No transport-cost calculator. No Agmarknet ingest. Instead: load the JSON on startup into an in-memory dict, serve season insights via REST, power the AI RecommendationCard via 3 parallel agents, stream the agent debate via WebSocket, and run the voice pipeline. The frontend already has RecommendationCard.tsx rendering recommendationLabel and recommendationRationale — your job is to make it real.

Updated voice stack (replacing Bhashini / Groq Whisper / AI4Bharat)

WhisperFlow STT → 3 parallel Llama 8B agents → ElevenLabs TTS (Multilingual v2). No 24h registration wait. No government API. ElevenLabs free tier is 10,000 chars/month — enough for demo. The 3-agent parallel path mirrors the dashboard debate, making voice feel like a first-class citizen of the same architecture, not a separate advisory shortcut.

Frontend state you are integrating against

19 components built and running. Routes: /, /dashboard, /commodity/[slug]. All data currently from lib/canned-data.ts — no API calls. Your integration target is lib/api.ts which currently only calls canned functions. TypeScript strict mode is on: every field you return must match the exact shape of CommodityInsightSummary or Member B gets type errors.

Core Agent Pipeline

Build the 3 analyst agents around season data

agents/season_optimist.py: Looks for price-above-MSP seasons, Kharif/Rabi uptrend signals, strongest price year. System prompt includes all 4 seasons as compact JSON.

agents/season_pessimist.py: Looks for price-below-MSP events, declining arrival volumes, glut signals across seasons.

All three use llama-3.1-8b-instant. Each returns: {"verdict":"HOLD"|"LEAN_SELL"|"DEFER"|"PROTECT","confidence":0-100,"reasoning":"...","key_seasons":["2024-25",...]}.
